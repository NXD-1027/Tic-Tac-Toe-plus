# v0.4 重新开始、悔棋与移动端优化实现文档

## 1. 版本目标

v0.4 的目标是在 v0.3 已经完成基础对局和胜负判定的基础上，补齐游戏控制功能，并优化移动端体验。

本阶段完成后，玩家应该可以随时重新开始游戏，可以撤销上一步操作，并且游戏页面在手机和平板上能够正常使用。

---

## 2. 本阶段要完成的内容

- 实现重新开始功能。
- 实现本地双人模式悔棋功能。
- 为后续人机模式悔棋预留结构。
- 优化按钮状态。
- 优化状态提示。
- 优化移动端布局。
- 优化触摸体验。
- 修复前几个版本中发现的交互问题。

## 3. 重新开始功能

### 3.1 功能要求

点击“重新开始”按钮后：

- 清空所有棋子。
- 重置所有小棋盘状态。
- 重置全局胜负状态。
- 当前玩家恢复为 X。
- `activeMicro` 恢复为 `null`。
- 清空 `moveHistory`。
- 页面重新渲染。

### 3.2 推荐实现

```javascript
function resetGame() {
  Object.assign(gameState, createInitialState());
  renderGame(gameState);
}
```

如果 `gameState` 是通过模块导出对象的方式共享，则可以使用 `Object.assign` 保持引用不变。

## 4. 悔棋功能

### 4.1 设计原则

悔棋的关键不是只删除棋子，还要恢复落子前的完整状态。

每次落子前，应在 `moveHistory` 中记录：

```javascript
{
  macroRow,
  macroCol,
  microIndex,
  player,
  previousActiveMicro,
  previousMicroWinner,
  previousGameWinner,
  previousGameOver
}
```

### 4.2 本地双人悔棋流程

悔棋时：

1. 如果 `moveHistory` 为空，直接返回。
2. 取出最后一步记录。
3. 清空对应格子。
4. 恢复该小棋盘之前的 `winner`。
5. 恢复 `activeMicro`。
6. 恢复 `winner`。
7. 恢复 `gameOver`。
8. 恢复 `currentPlayer` 为上一步落子的玩家。
9. 重新渲染页面。

### 4.3 推荐实现

```javascript
function undoMove(state) {
  if (state.moveHistory.length === 0) {
    return false;
  }

  const lastMove = state.moveHistory.pop();
  const microBoard = state.macroBoard[lastMove.macroRow][lastMove.macroCol];

  microBoard.cells[lastMove.microIndex] = null;
  microBoard.winner = lastMove.previousMicroWinner;

  state.activeMicro = lastMove.previousActiveMicro;
  state.winner = lastMove.previousGameWinner;
  state.gameOver = lastMove.previousGameOver;
  state.currentPlayer = lastMove.player;

  return true;
}
```

## 5. 人机模式悔棋预留

虽然 AI 在 v0.5 才正式实现，但 v0.4 可以先预留逻辑。

人机模式下，一次悔棋通常需要撤销两步：

```text
玩家的一步 + AI 的一步
```

建议写成：

```javascript
function undoForCurrentMode(state) {
  if (state.mode === "ai") {
    undoMove(state);
    undoMove(state);
    return true;
  }

  return undoMove(state);
}
```

后续 v0.5 实现 AI 后即可直接复用。

## 6. 按钮状态优化

### 6.1 重新开始按钮

重新开始按钮始终可用。

### 6.2 悔棋按钮

当 `moveHistory.length === 0` 时，悔棋按钮应禁用：

```javascript
undoButton.disabled = state.moveHistory.length === 0;
```

### 6.3 模式选择

如果当前已经有落子，可以在切换模式时弹出确认，或直接重置游戏。

建议初期采用简单方案：

```text
切换模式后自动重新开始游戏
```

## 7. 移动端布局优化

### 7.1 棋盘尺寸

棋盘宽度建议使用：

```css
.macro-board {
  width: min(92vw, 620px);
}
```

这样桌面端不会过大，移动端不会超出屏幕。

### 7.2 按钮布局

移动端下控制按钮可以换行：

```css
.control-panel {
  display: flex;
  flex-wrap: wrap;
  gap: 12px;
  justify-content: center;
}
```

### 7.3 触摸优化

建议为按钮和格子增加：

```css
button {
  touch-action: manipulation;
}

.cell {
  -webkit-tap-highlight-color: transparent;
}
```

这样可以减少移动端双击缩放和点击高亮带来的干扰。

## 8. 可访问性优化

可以为格子按钮增加 `aria-label`：

```javascript
cell.setAttribute(
  "aria-label",
  `大棋盘第 ${macroRow + 1} 行第 ${macroCol + 1} 列，小棋盘格子 ${microIndex + 1}`
);
```

按钮也应使用真实的 `<button>` 元素，而不是普通 `div`，这样键盘和读屏器支持更好。

## 10. 本阶段不做的内容

v0.4 暂不要求：

- 简单 AI。
- 中等 AI。
- Minimax。
- 音效。
- PWA。
- 自动化测试。

这些内容留到 v0.5 或后续版本。

## 11. 验收标准

完成 v0.4 后，应满足：

- [ ] 点击重新开始可以重置整局游戏。
- [ ] 重新开始后当前玩家恢复为 X。
- [ ] 重新开始后所有棋盘清空。
- [ ] 没有历史步骤时，悔棋按钮不可用或点击无效果。
- [ ] 本地双人模式下可以撤销上一步。
- [ ] 悔棋后棋子、当前玩家、目标小棋盘都恢复正确。
- [ ] 如果撤销的是导致胜利的一步，游戏结束状态能恢复。
- [ ] 手机宽度下页面不横向溢出。
- [ ] 手机端按钮容易点击。
- [ ] 浏览器控制台没有明显报错。
