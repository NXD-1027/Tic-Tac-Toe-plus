## 1. 项目概述

### 1.1 项目名称

**Tic-Tac-Toe Plus / 超级井字棋**

### 1.2 游戏简介

超级井字棋，也称 **Ultimate Tic-Tac-Toe**，是经典井字棋的进阶版本。

普通井字棋只有一个 3×3 棋盘，而超级井字棋由 9 个 3×3 小棋盘组成，整体形成一个 9×9 棋盘。玩家不仅要在局部小棋盘中取胜，还要通过赢下的小棋盘，在 3×3 的全局棋盘上形成三连。

这个规则使游戏具备更强的策略性：每一步落子不仅影响当前小棋盘，还会决定对手下一步必须前往哪个小棋盘。

### 1.3 项目目标

开发一个可直接在浏览器中运行的网页版超级井字棋游戏，目标如下：

- 支持本地双人对战。
- 支持人机对战，包括简单 AI 与中等难度 AI。
- 提供清晰的棋盘高亮和合法落子提示。
- 支持重新开始、悔棋、结果展示等基础功能。
- 界面适配桌面端与移动端。
- 项目可通过 GitHub Pages 部署为在线小游戏。

### 1.4 项目类型

- 单页静态网页应用。
- 不依赖后端服务器。
- 所有游戏逻辑在浏览器客户端完成。

## 2. 游戏规则

### 2.1 棋盘结构

游戏棋盘分为两层：

| 层级 | 说明 |
|---|---|
| 全局棋盘 / Macro Board | 一个 3×3 大棋盘，由 9 个小棋盘组成。 |
| 小棋盘 / Micro Board | 每个大格子内部是一个标准 3×3 井字棋。 |
| 总体棋盘 | 9×9 个可落子格子。 |

可以将整个游戏理解为：

```text
大棋盘：3 × 3
每个大格子：一个 3 × 3 小棋盘
整体棋盘：9 × 9
```

### 2.2 基本流程

1. 游戏开始时，玩家 X 先手。
2. 第一手可以在任意未结束的小棋盘内任意空格落子。
3. 之后，每一步落子的位置都会决定对手下一步必须进入哪个小棋盘。
4. 如果目标小棋盘已经结束，则对手可以自由选择任意仍在进行的小棋盘。
5. 玩家在小棋盘内形成三连，即赢下该小棋盘。
6. 玩家在全局棋盘中通过赢下的小棋盘形成三连，即赢得整局游戏。
7. 如果全局棋盘全部结束且无人三连，则整局为平局。

### 2.3 下一步棋盘规则

玩家落子的小格位置决定对手下一步的小棋盘位置。

例如：

- 玩家在某个小棋盘的左上角落子，对手下一步必须去全局棋盘的左上角小棋盘。
- 玩家在某个小棋盘的中心落子，对手下一步必须去全局棋盘的中心小棋盘。
- 玩家在某个小棋盘的右下角落子，对手下一步必须去全局棋盘的右下角小棋盘。

小棋盘内部索引可以设计为：

```text
0 | 1 | 2
---------
3 | 4 | 5
---------
6 | 7 | 8
```

对应到全局棋盘坐标：

```javascript
const targetRow = Math.floor(microIndex / 3);
const targetCol = microIndex % 3;
```

### 2.4 小棋盘胜负

小棋盘的胜负与普通井字棋相同：

- 横向三连获胜。
- 纵向三连获胜。
- 对角线三连获胜。
- 如果 9 个格子下满且无人三连，则该小棋盘为平局。

小棋盘一旦结束，就不能继续落子。

### 2.5 全局胜负

全局棋盘的胜负根据每个小棋盘的归属判断。

例如，如果玩家 X 赢下了左上、中上、右上三个小棋盘，那么 X 在全局棋盘上形成横向三连，获得整局胜利。

### 2.6 非法操作

以下行为应被判定为非法：

- 在已经有棋子的格子落子。
- 在已经结束的小棋盘落子。
- 在非当前指定的小棋盘落子。
- 游戏结束后继续落子。

非法点击应被忽略，并可通过界面提示告知玩家。

## 4. 技术选型

| 层次 | 技术方案 | 说明 |
|---|---|---|
| 页面结构 | HTML5 | 构建页面和棋盘容器。 |

| 逻辑 | JavaScript ES6+ | 管理游戏状态、事件和 AI。 |

## 7. 核心算法设计

### 7.1 胜利组合

小棋盘和全局棋盘都可以复用同一组胜利组合：

```javascript
const WIN_PATTERNS = [
  [0, 1, 2],
  [3, 4, 5],
  [6, 7, 8],
  [0, 3, 6],
  [1, 4, 7],
  [2, 5, 8],
  [0, 4, 8],
  [2, 4, 6]
];
```

### 7.2 胜负判断函数

```javascript
function checkWinner(cells) {
  for (const pattern of WIN_PATTERNS) {
    const [a, b, c] = pattern;

    if (cells[a] && cells[a] === cells[b] && cells[a] === cells[c]) {
      return cells[a];
    }
  }

  if (cells.every(Boolean)) {
    return "draw";
  }

  return null;
}
```

### 7.3 合法落子判断

```javascript
function isValidMove(state, macroRow, macroCol, microIndex) {
  if (state.gameOver) return false;

  const microBoard = state.macroBoard[macroRow][macroCol];

  if (microBoard.winner !== null) return false;
  if (microBoard.cells[microIndex] !== null) return false;

  if (state.activeMicro !== null) {
    return (
      state.activeMicro.row === macroRow &&
      state.activeMicro.col === macroCol
    );
  }

  return true;
}
```

### 7.4 下一步目标棋盘计算

```javascript
function getNextActiveMicro(state, microIndex) {
  const row = Math.floor(microIndex / 3);
  const col = microIndex % 3;
  const targetBoard = state.macroBoard[row][col];

  if (targetBoard.winner === null && targetBoard.cells.some(cell => cell === null)) {
    return { row, col };
  }

  return null;
}
```

### 7.5 主落子流程

```javascript
function makeMove(state, macroRow, macroCol, microIndex) {
  if (!isValidMove(state, macroRow, macroCol, microIndex)) {
    return false;
  }

  const microBoard = state.macroBoard[macroRow][macroCol];

  state.moveHistory.push({
    macroRow,
    macroCol,
    microIndex,
    player: state.currentPlayer,
    previousActiveMicro: state.activeMicro,
    previousMicroWinner: microBoard.winner,
    previousGameWinner: state.winner
  });

  microBoard.cells[microIndex] = state.currentPlayer;
  microBoard.winner = checkWinner(microBoard.cells);

  const globalCells = flattenMacroWinners(state.macroBoard);
  state.winner = checkWinner(globalCells);

  if (state.winner) {
    state.gameOver = true;
    return true;
  }

  state.activeMicro = getNextActiveMicro(state, microIndex);
  state.currentPlayer = state.currentPlayer === "X" ? "O" : "X";

  return true;
}
```

## 8. AI 设计

### 8.1 简单 AI

简单 AI 只需要从所有合法走法中随机选择一个。

实现步骤：

1. 扫描当前游戏状态。
2. 收集所有合法走法。
3. 随机选择其中一个。
4. 调用 `makeMove` 执行。

适合项目初期实现，用于验证人机对战流程是否正常。

### 8.2 中等难度 AI

中等难度 AI 可以基于启发式评分函数实现。

评分建议：

| 策略 | 分值建议 | 说明 |
|---|---:|---|
| 直接赢下全局棋盘 | +10000 | 最高优先级。 |
| 阻止玩家赢下全局棋盘 | +8000 | 防止对手获胜。 |
| 赢下当前小棋盘 | +1000 | 提高局部优势。 |
| 阻止玩家赢下小棋盘 | +700 | 防守关键局部。 |
| 占据中心小棋盘 | +100 | 中心位置战略价值较高。 |
| 占据小棋盘中心格 | +50 | 普通井字棋中中心位置更强。 |
| 把对手送进已结束棋盘 | +80 | 使对手自由选择，需要谨慎评分。 |
| 把对手送进其优势棋盘 | -200 | 避免送分。 |

后续可在此基础上加入深度 2 至 3 的 Minimax，并使用 Alpha-Beta 剪枝优化性能。

### 8.3 性能要求

- 简单 AI 应几乎立即响应。
- 中等 AI 单步计算建议控制在 200ms 内。
- 若 AI 计算变复杂，可以考虑使用 `setTimeout` 或 Web Worker 避免页面卡顿。

## 9. 界面设计

### 9.1 页面布局

页面可分为三部分：

```text
顶部：标题、当前玩家、下一步提示
中间：9×9 游戏棋盘

```

### 9.2 视觉设计建议

| 元素 | 设计建议 |
|---|---|
| 当前可落子小棋盘 | 使用高亮边框、发光效果或浅色背景。 |
| 不可落子区域 | 降低透明度或添加灰色遮罩。 |
| 已获胜小棋盘 | 显示大号 `X` 或 `O` 水印。 |
| 平局小棋盘 | 显示 `Draw` 或灰色标记。 |
| 当前玩家 | 使用醒目的状态栏提示。 |
| 游戏结束 | 弹窗或顶部提示显示胜者。 |

### 9.3 响应式设计

- 桌面端棋盘宽度建议 520px 至 640px。
- 移动端棋盘宽度建议使用 `min(92vw, 620px)`。
- 使用 CSS Grid 保证棋盘格子对齐。
- 手机端按钮应足够大，避免误触。

## 12. 部署方案

### 12.1 本地运行

项目为纯静态网页，开发完成后可以直接打开：

```text
index.html
```

也可以使用 VS Code 的 Live Server 插件启动本地服务。

### 12.2 GitHub Pages 部署

1. 将项目代码推送到 GitHub 仓库。
2. 打开仓库 `Settings`。
3. 进入 `Pages`。
4. Source 选择 `Deploy from a branch`。
5. Branch 选择 `main`。
6. 目录选择 `/root`。
7. 保存后等待部署完成。

部署成功后，访问地址类似：

```text
https://NXD-1027.github.io/Tic-Tac-Toe-plus/
```
