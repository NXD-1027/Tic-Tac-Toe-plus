# v0.5 AI 对战实现文档

## 1. 版本目标

v0.5 的目标是在本地双人对战、胜负判定、重新开始和悔棋功能稳定之后，加入人机对战模式。

本阶段完成后，玩家可以选择与 AI 对战。AI 至少支持两个难度：简单难度和中等难度。

---

## 2. 本阶段要完成的内容

- 实现人机对战模式。
- 实现简单 AI。
- 实现中等难度 AI。
- 实现 AI 自动落子流程。
- 确保 AI 只选择合法走法。
- 人机模式下支持重新开始。
- 人机模式下支持悔棋，一次撤销玩家和 AI 的两步。
- 优化 AI 思考时的界面提示。

---

## 3. 模式设计

游戏状态中使用 `mode` 区分模式：

```javascript
mode: "local" // 本地双人
mode: "ai"    // 人机对战
```

AI 难度使用 `aiDifficulty` 表示：

```javascript
aiDifficulty: "easy"   // 简单 AI
aiDifficulty: "medium" // 中等 AI
```

建议默认：

```javascript
mode: "local",
aiDifficulty: "easy"
```

---

## 4. 玩家与 AI 阵营

初期建议固定：

| 角色 | 棋子 |
|---|---|
| 玩家 | X |
| AI | O |

这样可以降低复杂度。后续版本再支持玩家选择先后手。

## 5. 合法走法收集

AI 的第一步不是思考策略，而是获取所有合法走法。

建议实现：

```javascript
function getLegalMoves(state) {
  const moves = [];

  for (let macroRow = 0; macroRow < 3; macroRow++) {
    for (let macroCol = 0; macroCol < 3; macroCol++) {
      for (let microIndex = 0; microIndex < 9; microIndex++) {
        if (isValidMove(state, macroRow, macroCol, microIndex)) {
          moves.push({ macroRow, macroCol, microIndex });
        }
      }
    }
  }

  return moves;
}
```

这个函数后续简单 AI 和中等 AI 都会使用。

---

## 6. 简单 AI

### 6.1 策略

简单 AI 从所有合法走法中随机选择一步。

### 6.2 实现示例

```javascript
function getRandomMove(state) {
  const legalMoves = getLegalMoves(state);

  if (legalMoves.length === 0) {
    return null;
  }

  const index = Math.floor(Math.random() * legalMoves.length);
  return legalMoves[index];
}
```

简单 AI 的作用：

- 验证人机模式流程。
- 验证 AI 是否遵守合法落子规则。
- 为中等 AI 提供兜底逻辑。

---

## 7. 中等难度 AI

### 7.1 策略思路

中等 AI 使用启发式评分函数，而不是一开始就使用完整 Minimax。

每个合法走法都会被模拟并评分，AI 选择分数最高的一步。

### 7.2 评分建议

| 策略 | 分值建议 | 说明 |
|---|---:|---|
| 直接赢下全局棋盘 | +10000 | 最高优先级。 |
| 阻止玩家赢下全局棋盘 | +8000 | 防止玩家下一步获胜。 |
| 赢下当前小棋盘 | +1000 | 建立局部优势。 |
| 阻止玩家赢下小棋盘 | +700 | 防止玩家拿下小棋盘。 |
| 占据中心小棋盘 | +120 | 中心小棋盘战略价值较高。 |
| 占据小棋盘中心格 | +60 | 普通井字棋中中心格较强。 |
| 占据角落格 | +30 | 角落通常优于边格。 |
| 把玩家送到已结束小棋盘 | +80 | 让对手进入自由选择，需要谨慎加分。 |
| 把玩家送到危险小棋盘 | -300 | 避免让玩家轻易赢下小棋盘。 |

## 8. 模拟走法

中等 AI 需要临时模拟走法，不能直接污染真实状态。

建议实现深拷贝：

```javascript
function cloneState(state) {
  return structuredClone(state);
}
```

如果需要兼容旧浏览器，可以使用：

```javascript
function cloneState(state) {
  return JSON.parse(JSON.stringify(state));
}
```

然后：

```javascript
function evaluateMove(state, move, aiPlayer, humanPlayer) {
  const cloned = cloneState(state);
  makeMove(cloned, move.macroRow, move.macroCol, move.microIndex);

  let score = 0;

  if (cloned.winner === aiPlayer) score += 10000;
  if (cloned.winner === humanPlayer) score -= 10000;

  return score;
}
```

注意：模拟时调用的 `makeMove` 不能触发真实 DOM 渲染，也不能触发 AI 自动继续落子。因此游戏逻辑和渲染逻辑必须分离。

## 9. 中等 AI 走法选择

```javascript
function getMediumMove(state, aiPlayer = "O", humanPlayer = "X") {
  const legalMoves = getLegalMoves(state);

  if (legalMoves.length === 0) {
    return null;
  }

  let bestMove = legalMoves[0];
  let bestScore = -Infinity;

  for (const move of legalMoves) {
    const score = evaluateMove(state, move, aiPlayer, humanPlayer);

    if (score > bestScore) {
      bestScore = score;
      bestMove = move;
    }
  }

  return bestMove;
}
```

如果多个走法分数相同，可以随机选择其中一个，避免 AI 行为过于固定。

## 10. AI 自动落子流程

玩家完成落子后，如果满足以下条件：

- 当前模式是 `ai`。
- 游戏没有结束。
- 当前玩家是 AI。

则触发 AI 落子。

建议加入短暂延迟，让玩家感觉 AI 在思考：

```javascript
function maybeTriggerAiMove() {
  if (gameState.mode !== "ai") return;
  if (gameState.gameOver) return;
  if (gameState.currentPlayer !== "O") return;

  setStatusText("AI 正在思考...");

  setTimeout(() => {
    const move = getAiMove(gameState);

    if (move) {
      makeMove(gameState, move.macroRow, move.macroCol, move.microIndex);
      renderGame(gameState);
    }
  }, 300);
}
```

## 11. AI 入口函数

`ai.js` 中建议统一提供：

```javascript
function getAiMove(state) {
  if (state.aiDifficulty === "medium") {
    return getMediumMove(state, "O", "X");
  }

  return getRandomMove(state);
}
```

这样 UI 层不用关心具体 AI 策略。

## 12. 人机模式悔棋

人机模式下建议一次撤销两步：

1. 撤销 AI 的上一步。
2. 撤销玩家的上一步。

伪代码：

```javascript
function undoForCurrentMode(state) {
  if (state.mode === "ai") {
    undoMove(state);
    undoMove(state);
    return true;
  }

  return undoMove(state);
}
```

需要注意：

- 如果 AI 还没落子，只撤销一步即可。
- 如果游戏刚开始，没有历史记录，则不执行。
- 撤销后应轮到玩家 X。

## 13. UI 调整

本阶段需要增加：

- 模式选择：本地双人 / 人机对战。
- AI 难度选择：简单 / 中等。
- AI 思考中提示。
- AI 落子期间禁止玩家连续点击。

可加入状态字段：

```javascript
isAiThinking: false
```

当 AI 思考时：

- 禁止玩家点击棋盘。
- 显示提示 `AI 正在思考...`。
- AI 落子后恢复正常。

## 14. 本阶段不做的内容

v0.5 暂不要求：

- 完整 Minimax。
- Alpha-Beta 剪枝。
- 在线多人对战。
- 服务器存档。
- 复杂 AI 难度。
- 玩家选择先后手。

这些可以作为后续版本扩展。
