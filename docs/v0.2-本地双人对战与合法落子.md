# v0.2 本地双人对战与合法落子实现文档

## 1. 版本目标

v0.2 的目标是在 v0.1 棋盘渲染的基础上，实现超级井字棋的基础对局流程。

本阶段完成后，玩家 X 和 O 可以轮流点击格子落子，并且系统能够根据超级井字棋规则判断当前点击是否合法。

---

## 2. 本阶段要完成的内容

- 初始化完整游戏状态 `gameState`。
- 实现本地双人轮流落子。
- 实现已占用格子不可重复落子。
- 实现已结束小棋盘不可落子。
- 实现 `activeMicro` 规则。
- 实现自由选择模式。
- 更新当前玩家显示。
- 更新下一步目标小棋盘提示。

---

## 3. 核心规则回顾

超级井字棋的关键规则是：

1. 第一手可以在任意小棋盘落子。
2. 玩家落子的小格位置决定对手下一步必须进入哪个小棋盘。
3. 如果目标小棋盘已经结束，则下一位玩家可以自由选择任意未结束小棋盘。
4. 已有棋子的格子不能再次落子。
5. 已结束的小棋盘不能继续落子。

---

## 4. 游戏状态设计

`game.js` 中应维护一个完整的游戏状态对象：

```javascript
const gameState = {
  macroBoard: [],
  currentPlayer: "X",
  activeMicro: null,
  gameOver: false,
  winner: null,
  moveHistory: [],
  mode: "local",
  aiDifficulty: "easy"
};
```

### 字段说明

| 字段 | 说明 |
|---|---|
| `macroBoard` | 3×3 大棋盘，每个元素是一个小棋盘。 |
| `currentPlayer` | 当前玩家，值为 `X` 或 `O`。 |
| `activeMicro` | 当前必须落子的小棋盘坐标，`null` 表示自由选择。 |
| `gameOver` | 游戏是否结束。 |
| `winner` | 全局赢家。 |
| `moveHistory` | 走法历史，后续用于悔棋。 |
| `mode` | 游戏模式，本阶段固定为 `local`。 |
| `aiDifficulty` | AI 难度，本阶段暂不使用。 |

## 5. 小棋盘结构

每个小棋盘对象建议如下：

```javascript
function createMicroBoard() {
  return {
    cells: Array(9).fill(null),
    winner: null
  };
}
```

`winner` 在 v0.2 中可以先保留字段，不一定完整实现胜负判定。v0.3 再正式更新它。

## 7. 合法落子判断

核心函数：

```javascript
function isValidMove(state, macroRow, macroCol, microIndex) {
  if (state.gameOver) return false;

  const microBoard = state.macroBoard[macroRow][macroCol];

  if (!microBoard) return false;
  if (microBoard.winner !== null) return false;
  if (microBoard.cells[microIndex] !== null) return false;

  if (state.activeMicro !== null) {
    return (
      state.activeMicro.row === macroRow &&
      state.activeMicro.col === macroCol
    );
  }

  return true;
}
```

---

## 8. 下一步目标小棋盘

玩家落子位置 `microIndex` 对应下一位玩家必须进入的小棋盘。

```javascript
function getNextActiveMicro(state, microIndex) {
  const row = Math.floor(microIndex / 3);
  const col = microIndex % 3;
  const targetBoard = state.macroBoard[row][col];

  if (targetBoard.winner === null && targetBoard.cells.some(cell => cell === null)) {
    return { row, col };
  }

  return null;
}
```

当返回 `null` 时，表示下一位玩家可以自由选择。

## 9. 落子函数

本阶段的 `makeMove` 主要负责：

1. 判断是否合法。
2. 写入当前玩家棋子。
3. 记录历史。
4. 计算下一步目标小棋盘。
5. 切换玩家。

```javascript
function makeMove(state, macroRow, macroCol, microIndex) {
  if (!isValidMove(state, macroRow, macroCol, microIndex)) {
    return false;
  }

  const microBoard = state.macroBoard[macroRow][macroCol];

  state.moveHistory.push({
    macroRow,
    macroCol,
    microIndex,
    player: state.currentPlayer,
    previousActiveMicro: state.activeMicro
  });

  microBoard.cells[microIndex] = state.currentPlayer;
  state.activeMicro = getNextActiveMicro(state, microIndex);
  state.currentPlayer = state.currentPlayer === "X" ? "O" : "X";

  return true;
}
```

## 10. 渲染更新

`render.js` 需要根据状态更新：

- 每个格子的 X / O。
- 当前玩家文本。
- 下一步提示文本。
- 当前可落子小棋盘高亮。
- 不可落子区域样式。

提示文字示例：

```text
当前玩家：X
下一步：可以自由选择任意未结束小棋盘
```

或者：

```text
当前玩家：O
下一步：必须在第 2 行第 3 列的小棋盘落子
```

## 11. CSS 类名建议

```text

.micro-board.free
.cell.x
.cell.o

```

说明：

| 类名 | 用途 |
|---|---|
| `active` | 当前必须落子的小棋盘。 |
| `inactive` | 当前不能落子的小棋盘。 |
| `free` | 自由选择模式下可落子的小棋盘。 |
| `x` | X 棋子样式。 |
| `o` | O 棋子样式。 |
| `disabled` | 不可点击格子。 |

## 12. 本阶段不做的内容

v0.2 暂不要求：

- 小棋盘胜负判定。
- 全局胜负判定。
- 获胜小棋盘大号水印。
- 悔棋完整恢复。
- AI。
- 动画音效。

但代码结构要为这些功能预留位置。
