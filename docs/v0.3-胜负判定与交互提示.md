# v0.3 胜负判定与交互提示实现文档

## 1. 版本目标

v0.3 的目标是在 v0.2 已经可以正常轮流落子的基础上，加入完整的胜负判定和更清晰的交互提示。

本阶段完成后，游戏应能够判断每个小棋盘的胜负或平局，并能根据已经赢下的小棋盘判断全局胜负。

---

## 2. 本阶段要完成的内容

- 实现小棋盘胜负判定。
- 实现全局棋盘胜负判定。
- 实现小棋盘平局判定。
- 实现全局平局判定。
- 在界面上标记已被 X 或 O 赢下的小棋盘。
- 在界面上标记平局小棋盘。
- 游戏结束后显示胜者或平局结果。
- 游戏结束后禁止继续落子。
- 强化当前可落子区域提示。

---

## 3. 胜利组合

小棋盘和全局棋盘都使用标准井字棋的 8 种胜利组合：

```javascript
const WIN_PATTERNS = [
  [0, 1, 2],
  [3, 4, 5],
  [6, 7, 8],
  [0, 3, 6],
  [1, 4, 7],
  [2, 5, 8],
  [0, 4, 8],
  [2, 4, 6]
];
```

---

## 4. 胜负判断函数

建议在 `game.js` 中实现通用函数：

```javascript
function checkWinner(cells) {
  for (const pattern of WIN_PATTERNS) {
    const [a, b, c] = pattern;

    if (cells[a] && cells[a] !== "draw" && cells[a] === cells[b] && cells[a] === cells[c]) {
      return cells[a];
    }
  }

  if (cells.every(Boolean)) {
    return "draw";
  }

  return null;
}
```

注意：

- 小棋盘的 `cells` 中只会出现 `X`、`O`、`null`。
- 全局棋盘的 `cells` 中可能出现 `X`、`O`、`draw`、`null`。
- 判断全局胜负时，`draw` 不能参与三连获胜。

---

## 5. 小棋盘胜负更新

每次落子后，需要立即检查当前小棋盘：

```javascript
const microWinner = checkWinner(microBoard.cells);

if (microWinner) {
  microBoard.winner = microWinner;
}
```

可能结果：

| 返回值 | 含义 |
|---|---|
| `X` | X 赢下该小棋盘。 |
| `O` | O 赢下该小棋盘。 |
| `draw` | 小棋盘平局。 |
| `null` | 小棋盘仍在进行。 |

---

## 6. 全局棋盘胜负更新

先把 3×3 小棋盘赢家转换成一维数组：

```javascript
function getGlobalCells(macroBoard) {
  return macroBoard.flat().map(board => board.winner);
}
```

然后复用 `checkWinner`：

```javascript
const globalCells = getGlobalCells(state.macroBoard);
const globalWinner = checkWinner(globalCells);

if (globalWinner) {
  state.winner = globalWinner;
  state.gameOver = true;
}
```

## 7. 更新 makeMove 流程

v0.3 之后，`makeMove` 的流程应为：

1. 判断落子是否合法。
2. 记录历史。
3. 写入当前玩家棋子。
4. 判断当前小棋盘胜负。
5. 判断全局棋盘胜负。
6. 如果游戏结束，停止后续切换。
7. 否则计算下一步目标小棋盘。
8. 切换当前玩家。
9. 返回成功。

示例结构：

```javascript
function makeMove(state, macroRow, macroCol, microIndex) {
  if (!isValidMove(state, macroRow, macroCol, microIndex)) {
    return false;
  }

  const microBoard = state.macroBoard[macroRow][macroCol];

  state.moveHistory.push({
    macroRow,
    macroCol,
    microIndex,
    player: state.currentPlayer,
    previousActiveMicro: state.activeMicro,
    previousMicroWinner: microBoard.winner,
    previousGameWinner: state.winner,
    previousGameOver: state.gameOver
  });

  microBoard.cells[microIndex] = state.currentPlayer;
  microBoard.winner = checkWinner(microBoard.cells);

  const globalWinner = checkWinner(getGlobalCells(state.macroBoard));

  if (globalWinner) {
    state.winner = globalWinner;
    state.gameOver = true;
    return true;
  }

  state.activeMicro = getNextActiveMicro(state, microIndex);
  state.currentPlayer = state.currentPlayer === "X" ? "O" : "X";

  return true;
}
```

## 8. 交互提示设计

### 8.1 当前玩家提示

正常对局时：

```text
当前玩家：X
```

游戏结束时：

```text
游戏结束：X 获胜
```

或：

```text
游戏结束：平局
```

### 8.2 下一步目标提示

自由选择模式：

```text
下一步：可以自由选择任意未结束小棋盘
```

指定小棋盘模式：

```text
下一步：必须在第 2 行第 3 列的小棋盘落子
```

游戏结束时：

```text
本局已结束，请点击重新开始
```

## 9. 小棋盘状态样式

建议增加以下 CSS 类：

```text
.micro-board.won-x
.micro-board.won-o
.micro-board.draw
.micro-board.active
.micro-board.inactive
```

### 9.1 已获胜小棋盘

已被 X 赢下：

```html
<div class="micro-board won-x"></div>
```

已被 O 赢下：

```html
<div class="micro-board won-o"></div>
```

### 9.2 平局小棋盘

```html
<div class="micro-board draw"></div>
```

可以通过伪元素显示水印：

```css
.micro-board.won-x::after {
  content: "X";
}

.micro-board.won-o::after {
  content: "O";
}

.micro-board.draw::after {
  content: "平";
}
```

## 10. 本阶段不做的内容

v0.3 暂不要求：

- 悔棋完整功能。
- AI。
- 音效。
- 复杂动画。
- 部署。

但本阶段应确保状态记录足够支持 v0.4 的悔棋。

## 11. 验收标准

完成 v0.3 后，应满足：

- [ ] 小棋盘横向三连可以正确判胜。
- [ ] 小棋盘纵向三连可以正确判胜。
- [ ] 小棋盘对角线三连可以正确判胜。
- [ ] 小棋盘下满无人胜利时判为平局。
- [ ] 已结束小棋盘不能继续落子。
- [ ] 全局横向三连可以正确判胜。
- [ ] 全局纵向三连可以正确判胜。
- [ ] 全局对角线三连可以正确判胜。
- [ ] 全局平局可以正确显示。
- [ ] 游戏结束后不能继续落子。
- [ ] 获胜或平局的小棋盘有明显标识。
- [ ] 当前可落子小棋盘高亮正确。
- [ ] 浏览器控制台没有明显报错。
